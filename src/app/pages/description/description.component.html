The technical exercise

Summary

This is an Angular project created via Angular Cli. It uses all standard functionality including Lint validation and Unit tests.
All requirements from the description of the exercise have been replied.

Intent

In this development, I tried to make a finished project ready to deploy. My main focus was on an overall overview with all design details, project structure, and scroll/pagination functionality.
From a development process perspective, I decided to refuse using other common functionality (like e.g. translations, authentication, shared services with a state storing etc.) to avoid extending this task and be able to complete it at time.

Images source

I used https://picsum.photos/ as a source for images. It supports pagination as well.

Known issues

On certain view dimensions, you can see that full-width alignment of the latest row looks not appropriate.
In the real application, I would suggest finding a proper way of using images (by changing the technical task), or either create an auto-alignment function where you will check the number of items fit the row and depends on this set row/column parameters. But it sounds quite "hacky", so I decided to follow the initial task and solve it with css.

Because of loading a lot of high-resolution images your browsers might lag and skip some animations.
In the real application, I would suggest using minified images for the preview page.

Project structure

assets/sass

I used bootstrap and common styles. All of them depend on a group of variables that can be included in components as well.

Entities

General structuring was done "by feature" with "shared" folder using. A good example of this is "pages/list".
